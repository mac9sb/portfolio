Full stack safety with Hummingbird

<div class="markdown-content"><h1>Full Stack Safety with Hummingbird</h1><p>Hummingbird is a lightweight, flexible Swift web framework that enables type-safe full-stack development. This article explores how to leverage Swift’s type system across your entire stack.</p><h2>Why Hummingbird?</h2><p>Hummingbird offers several advantages:</p><ul><li><p><strong>Lightweight</strong> - Minimal overhead and fast startup</p></li><li><p><strong>Async/await native</strong> - Built for Swift concurrency</p></li><li><p><strong>Modular</strong> - Use only what you need</p></li><li><p><strong>Type-safe</strong> - Leverage Swift’s strong typing</p></li></ul><h2>Shared Models</h2><p>The key to full-stack type safety is sharing models between frontend and backend:</p><pre class="markdown-code-block language-swift"><code>// Shared/Sources/Models/User.swift public struct User: Codable, Sendable { public let id: UUID public let email: String public let name: String public let createdAt: Date public init(id: UUID, email: String, name: String, createdAt: Date) { self.id = id self.email = email self.name = name self.createdAt = createdAt } } </code></pre><h2>Backend Implementation</h2><p>Setting up routes with Hummingbird:</p><pre class="markdown-code-block language-swift"><code>import Hummingbird func buildApplication() -&gt; some ApplicationProtocol { let router = Router() router.get(&quot;/api/users/:id&quot;) { request, context -&gt; User in guard let id = request.parameters.get(&quot;id&quot;, as: UUID.self) else { throw HTTPError(.badRequest) } return try await UserService.find(id: id) } router.post(&quot;/api/users&quot;) { request, context -&gt; User in let dto = try await request.decode(as: CreateUserDTO.self, context: context) return try await UserService.create(dto) } return Application(router: router) } </code></pre><h2>Type-Safe API Client</h2><p>The same models work on the client side:</p><pre class="markdown-code-block language-swift"><code>// iOS/Sources/APIClient.swift actor APIClient { private let baseURL: URL private let decoder = JSONDecoder() func fetchUser(id: UUID) async throws -&gt; User { let url = baseURL.appending(path: &quot;/api/users/\(id)&quot;) let (data, _) = try await URLSession.shared.data(from: url) return try decoder.decode(User.self, from: data) } } </code></pre><h2>Validation</h2><p>Type-safe validation using Swift’s type system:</p><pre class="markdown-code-block language-swift"><code>struct CreateUserDTO: Codable, Validatable { let email: String let name: String let password: String func validate() throws { guard email.contains(&quot;@&quot;) else { throw ValidationError.invalidEmail } guard password.count &gt;= 8 else { throw ValidationError.passwordTooShort } } } </code></pre><h2>Benefits Realized</h2><p>In production, this approach has delivered:</p><ul><li><p><strong>Compile-time error detection</strong> for API mismatches</p></li><li><p><strong>Reduced runtime errors</strong> by 80%</p></li><li><p><strong>Faster development cycles</strong> due to shared code</p></li><li><p><strong>Better documentation</strong> through types</p></li></ul><h2>Conclusion</h2><p>Hummingbird enables true full-stack Swift development with type safety throughout. By sharing models between frontend and backend, you eliminate an entire class of bugs and improve developer experience.</p></div>
